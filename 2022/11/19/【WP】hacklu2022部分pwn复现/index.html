分析

本地启动服务

服务在4444端口暴露三个命令：

  • S：在内存中存储键值对
    • 键必须是12字节长
    • 值只能包含由bytes.fromhex(data)处理的十六进制字符
  • D：将存储的数据转储到磁盘
    • 保存内存中的entries到storage目录中
    • 键作为文件名
    • 值作为文件内容
  • P：在数据上运行插件代码
    • 通过在plugins目录中传递文件名来运行plugin
    • 将插件文件解释为python字节码并调用exec
    • 使内存中的数据作为已编译CodeType的co_consts可用

编写exploit字节码

WITH_EXCEPT_START
Calls the function in position 7 on the stack with the top three items on the stack as arguments. Used to implement the call context_manager.exit(*exc_info()) when an exception has occurred in a with statement.

调用在堆栈上位置7的函数，使用堆栈最上面的三个项作为参数。用于在with语句中发生异常时实现context_manager._exit_(*exc_info())调用

New in version 3.9.

这意味着我们找到了一个带有一些限制的CALL_FUNCTION操作，题目也很贴心地给出了一个调试功能

并且这个函数正好接受三个参数，它们可以和WITH_EXCEPT_START一起传递。因为所有数据条目都是通过co_const传递给插件代码的，所以可以将它们作为参数引用，只要它们在我们可以访问的边界内（即索引0x30-0x39表示”0”-“9”，索引0x61-0x66表示’a’-‘f’）

所以我们可以用这个函数将真正的不受限制的利用字节码作为存储的键，并将这些写入另一个插件文件，这将是我们最终利用的plugin

所以我们现在可以生成调用函数co_const[func_index]的字节码，参数co_const[content_index]和co_consts[filename_index]
