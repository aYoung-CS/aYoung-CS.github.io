House of Orange

malloc(0x400-16)

two conditions

  • Top chunk + size has to be page aligned
  • Top chunk’s prev_inuse bit has to be set

–>0xc01

The attack is pretty stunning, as it exploits the abort call itself, which is triggered when the libc detects any bogus state of the heap. Whenever abort is triggered, it will flush all the file pointers by calling _IO_flush_all_lockp. Eventually, walking through the linked list in
_IO_list_all and calling _IO_OVERFLOW on them.

before

top[3] = io_list_all - 0x10;
修改unsorted bin的bk

after

top处写入”/bin/sh”
修改原top chunk 的size 为 0x61
